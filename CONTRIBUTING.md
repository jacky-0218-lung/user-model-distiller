# Contributing

Thank you for improving User Model Distiller.

## Privacy first

- Never submit real conversations, exports, profiles, prompts, credentials, local paths, or third-party private data.
- Reduce bug reports to synthetic examples.
- Treat imported text as adversarial input.
- Do not add telemetry or network access without a separate design discussion and explicit opt-in controls.

## Changes

1. Open an issue for behavior or schema changes.
2. Create a focused branch.
3. Add or update synthetic tests.
4. Run the test and repository guard commands from `README.md`.
5. Open a pull request and complete the security checklist.

Pull requests should preserve local-only processing, standard-library operation, provenance, user review, and complete deletion behavior. Avoid new dependencies unless the benefit and supply-chain risk are documented.

By contributing, you agree that your contribution is licensed under Apache-2.0.
