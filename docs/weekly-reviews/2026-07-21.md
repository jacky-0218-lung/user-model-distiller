## 2. 本週研究發現與來源

### 方向 A — Skill 撰寫技巧與平台變化
- Anthropic 官方《Skill authoring best practices》要點:`name` ≤ 64 字元、`description` ≤ 1024 字元且需含「做什麼 + 何時用」;SKILL.md 主體建議 < 500 行;參考檔只往下**一層**;超過 100 行的參考檔應含目錄;發佈前至少 3 個評估案例;**避免平台專屬路徑/語法**(例:Windows 反斜線路徑)、術語全文一致、避免時間敏感資訊。
  - 來源:https://platform.claude.com/docs/en/agents-and-tools/agent-skills/best-practices
- Codex changelog 顯示近期在 skills/AGENTS.md 的變更集中在**載入效能與授權委派**(如平行化技能載入、AGENTS.md 可回應環境變化、允許 AGENTS.md/skills 授權 delegation),以及跨 session 的 world-state 持久化;對本 repo 的核心格式無破壞性影響,格式慣例維持穩定。
  - 來源:https://learn.chatgpt.com/docs/changelog

### 方向 B — 競品與類似工具(memory / personalization)
- 2026 年 AI agent memory 框架比較(mem0 / Letta / Zep / Cognee / Supermemory)可借鏡的設計:
  - **時間有效性視窗(Zep)**:記錄偏好「何時成立、何時改變」,而非只記當前值 → 本 repo 已有 `observation dates` 與 `supersede`,方向一致。
  - **顯式遺忘(Supermemory)**:多數框架把刪除當邊緣案例;本 repo 已把 forget/superseded 當一等公民,並在 `evaluation.md` 有「刪除評估」切片,屬領先實務。
  - **本機優先離線(Cognee)**:與本 repo 的 local-first 定位相符,佐證產品方向。
  - 來源:https://atlan.com/know/best-ai-agent-memory-frameworks-2026/ 、https://www.cognee.ai/blog/guides/best-ai-memory-layers-for-ai-agents-in-2026-comparison

### 方向 C — 安全與隱私強化
- OWASP/業界 2026 觀察:prompt injection 仍是 agentic AI 生產環境的**首要**失敗來源;縱深防禦、把外部內容視為不可信、最小揭露仍是主軸。
  - 本 repo 已把 transcript/工具輸出視為不可信、編譯後 runtime view 不含來源引文以降低持續性注入,方向正確,本週無需變更。
  - 來源:https://www.helpnetsecurity.com/2026/06/11/owasp-prompt-injection-ai-security-failures/ 、https://cheatsheetseries.owasp.org/cheatsheets/AI_Agent_Security_Cheat_Sheet.html
